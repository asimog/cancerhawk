Start the backend

GitHub Pages is static, so it cannot run Python by itself. Start CancerHawk locally, then this page will display the backend here.

uv run --with-requirements app/requirements.txt --with-requirements app/requirements-dev.txt python -m app.main

Once the backend is running, paste your OpenRouter API key, choose models, and run a fresh block. Completed blocks publish into results/block-N/. If you enable git publishing locally, the backend can commit and push the generated block to GitHub Pages.
